# Async Report Generator (черновик)

Сервис для асинхронной генерации отчетов в форматах Excel и PDF. Поддерживает два режима работы: выгрузка данных из базы по сохраненному шаблону, либо форматирование переданных пользователем данных.

## Функционал

### 1. Управление шаблонами отчетов

Шаблон - это сохраненная конфигурация, описывающая структуру и оформление отчета.

Поля шаблона:
- `id`, `name`, `description`
- `source_type`: `database` или `inline`
- `source_config`: для `database` - таблица и список полей
- `columns`: массив колонок (поле -> заголовок)
- `default_filters`: предустановленные фильтры для `database`
- `output_format`: `excel` или `pdf`
- `styling`: настройки оформления (опционально)

Эндпоинты:
- `POST /api/v1/templates` - создать шаблон
- `GET /api/v1/templates` - список шаблонов
- `GET /api/v1/templates/{id}` - получить шаблон
- `PUT /api/v1/templates/{id}` - обновить шаблон
- `DELETE /api/v1/templates/{id}` - удалить шаблон

### 2. Создание задачи на генерацию отчета

Два режима создания:

#### Режим A: По шаблону из БД

Указывается `template_id`, диапазон дат и опционально переопределяются фильтры. Воркер сам собирает данные из БД.

```json
{
  "template_id": "uuid",
  "date_from": "2026-01-01",
  "date_to": "2026-01-31",
  "override_filters": {
    "status": "completed"
  }
}
```

#### Режим B: С переданными данными (inline)

Пользователь передает готовый массив данных и опционально ссылается на шаблон для оформления.

```json
{
  "name": "Кастомный отчет",
  "template_id": "uuid",
  "output_format": "excel",
  "columns": [
    {"field": "name", "header": "Имя"},
    {"field": "amount", "header": "Сумма"}
  ],
  "data": [
    {"name": "Иванов", "amount": 1000},
    {"name": "Петров", "amount": 2500}
  ]
}
```

Эндпоинты:
- `POST /api/v1/reports` - создать задачу на генерацию

### 3. Статусы задачи

- `pending` - задача создана, ожидает в очереди
- `processing` - воркер обрабатывает задачу
- `completed` - файл готов и загружен в S3
- `failed` - ошибка при генерации

### 4. Проверка статуса и скачивание

Эндпоинты:
- `GET /api/v1/reports` - список задач пользователя с пагинацией
- `GET /api/v1/reports/{task_id}` - получить статус и метаданные задачи
- `GET /api/v1/reports/{task_id}/download` - получить временную ссылку на скачивание

### 5. Асинхронная обработка

- При создании задачи Use Case сохраняет её в БД со статусом `pending`
- В RabbitMQ отправляется сообщение с `task_id`
- Celery-воркер получает `task_id`, меняет статус на `processing`
- В зависимости от типа задачи:
  - Database: выполняет запрос к БД с учетом фильтров и диапазона дат
  - Inline: использует переданный массив `data`
- Строит файл нужного формата (Excel или PDF)
- Загружает файл в MinIO, обновляет статус на `completed` и сохраняет `file_key`
- В случае ошибки статус становится `failed`, заполняется `error_message`

### 6. Безопасность и валидация

- Белый список таблиц и полей: для `database` шаблонов `source_config.table` и `fields` сверяются с разрешенными значениями
- Ограничение размера inline-данных: максимум 10000 строк, проверяется на уровне Pydantic
- Аутентификация: заголовок `X-User-Id`
- Rate Limiting: не более 5 запросов на создание задачи в минуту с одного `user_id` (Redis)

### 7. Хранение данных

PostgreSQL:
- `report_templates` - шаблоны
- `reports` - задачи генерации

Redis:
- Кеш статусов задач (TTL 60s)
- Rate Limiting
- Блокировки для предотвращения дублирования обработки

aioboto3 (S3):
- Бакет `reports`
- Ключ объекта: `{user_id}/{year}/{month}/{task_id}.{ext}`
- Pre-signed URL для скачивания (действует 1 час)

### 8. Мониторинг

Prometheus Metrics:
- `report_requests_total` - счетчик задач по статусам
- `report_generation_duration_seconds` - гистограмма времени генерации
- `celery_queue_length` - длина очереди RabbitMQ
- `s3_upload_errors_total` - ошибки загрузки в S3

Grafana Dashboard:
- RPS создания задач
- Загрузка очереди и количество воркеров
- Распределение задач по типам источников (database vs inline)

Логирование:
- Структурированные JSON-логи с `correlation_id`
- Возможность поиска по `report_id`
